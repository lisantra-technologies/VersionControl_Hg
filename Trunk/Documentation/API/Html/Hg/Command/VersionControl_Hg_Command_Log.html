Implements the log command.

The hg command-line client also uses 'history' as an alias. *

  1.  include_once 'VersionControl/Hg.php';
  2.  $hg = new VersionControl_Hg('/path/to/repository');
  3.  $hg->log()->run('verbose');

Show only log entries for the following dates:

  1.  $hg->log()->on('Dec 27, 2010')->run();
  2.  $hg->log()->before('Dec 27, 2010')->run();
  3.  $hg->log()->after('Dec 27, 2010')->run();
  4.  $hg->log()->between('Dec 27, 2010', '2010-12-31')->run();

Show only log entries for revision 2:

  1.  $hg->log()->revision('2')->format('raw')->run('verbose');
  2.  $hg->log()->revision('2')->run('verbose');

Show only log entries for specific files:

  1.  $hg->log()->files(array('index.php'))->run();

Exclude log entries for certain files:

  1.  $hg->log()->excluding('**.php')->run();

This displays nothing; excluding takes precedence. Remove excluding() to get only index.php:

  1.  $hg->log()->files(array('index.php'))->excluding('**.php')->run();

Include specific files, which might not show up otherwise. Using this with files can lead to unexpected results:

  1.  $hg->log()->including('**.php')->run();

Patching and diffing options are not available in this class as they might be when using the command line Mercurial client.
