Handling the Data Returned by Commands

Formatting Provider

Output formatting is provided by VersionControl/Hg/Command/Output/Formatter.php

Format: PHP Array

The default return format is as an array of data. Each item in the array correpsond to the lines of Mercurial's output. Each item is itself an array. The key is most often a numerical index corresponding to the output's line number. each key in the sub-array is a column name while the value is the data returned by mercurial.

This is the default; No argument to to the final run() method is needed to get data returned as arrays.

Format: Raw Text

Raw text is gotten by passing raw to the final run() method.

This text is assembled from the array returned by default. The lines are delimited by PHP_EOL and are thus platform independent. For example, if you run this code on Windows, your lines will be delmimited with /r/n.

<?php
$hg->log()->run('raw');
?>

or

<?php
$hg->log()->run('text');
?>

Format: JSON

JSON output is a string representation of the default array output.

JSON formatted output is gotten by passing json to the final run() method.

<?php
$hg->log()->run('json');
?>

Format: YAML

YAML output is a string representation of the default array output.

YAML formatted output is gotten by passing yaml to the final run() method.

<?php
$hg->log()->run('yaml');
?>

Format: PHP Serialized

This format is what appears when PHP serializes a variable.

Serialize formatted output is gotten by passing serialize to the final run() method.

<?php
$hg->log()->run('serialize');
?>

Format: XML

XML output is not yet supported.

Part of the problem is that aside from hg log, there really is no agreed upon XML vocabulary for version control. I see a promising opportunity for the enterprising programmer if I don't get to it first.
